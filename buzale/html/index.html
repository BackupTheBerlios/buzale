BuchstabenZahlenLernen (BuZaLe)

Einführung    Spielaufbau    Installation    Konfiguration

Einführung

 BuZaLe - der Titel ist gleich Programm.
Dieses Spiel dient dazu Kindern, begleitend zur Schule, die Buchstaben und Zahlen näher zu bringen.
Der Spielablauf ist hinsichtlich der gestellten Aufgaben frei konfigurierbar und kann dem aktuellen Wissensstand angepaßt werden.
Die zu lösenden Aufgaben werden in Form von Bild.- und Sounddateien vorgegeben. Dadurch ist es möglich eigene Bilder zusammen mit dem Kind auszusuchen, mit einer individuellen Sounddatei zu versehen und dann in das Spiel einzubinden.
Persönliches Das Spiel habe ich zusammen mit meiner Tochter (1. Schuljahr) entworfen. Ziel war es das Interesse am Lernen auf spielerische weise zu fördern. Die Bilder für die verschiedenen Aufgaben sowie die dazugehörige Sounddatei wurden von ihr mit Begeisterung zusammengestellt. Das Erlernen von Wörtern und Lösen von keinen Rechenaufgaben geschah dadurch weniger durch das eigentliche Spielen, als durch das kreative Erstellen der Spielaufgaben.
Während des Spiels können die Eingaben sowohl über die Maus (klick auf die entsprechenden Buchstaben oder auf die entsprechenden Zahlen) als auch über die Tastatur erfolgen.
Die Farbe und Größe der Zeichen auf dem Bildschirm können individuell angepaßt werden.

Spielaufbau

 Der Spielaufblauf besteht aus folgenden Spielstufen:
  • Nach dem Progammstart befindet man sich im Level 'START'
    Hier werden zyklisch alle für diese Spielstufe definierten Bilder angezeigt und die dazugehörende Sounddatei abgespielt.
    Durch Anwahl der Taste Deutsch/Mathe verzweigt man in die entsprechende Spielstufe DA bzw. MA.
  • In den Spielstufen DA..DE bzw. MA..ME werden die Bilder mit den gestellten Aufgaben in zufälliger Reihenfolge angezeigt. Durch die Eingabe der richtigen Antwort, wird die nächste Aufgabe ausgewählt. Der Name der dargestellten Bilddatei ist gleichzeitig die richtige Antwort. So können im Prinzip x-beliebige Aufgaben als Bilddatei erzeugt und mit dem Lösungstext als Filename in das Spiel eingebunden werden. Eine normale Spielstufe ist beendet wenn 20 Aufgaben richtig gelöst wurden.
    Die Anzahl der Stufen bis zum Level 'ENDE' kann parametriert werden.
    Das letzte Level (DE bzw. ME) kann als sogenanntes Bonus-Level gekennzeichnet werden. Dort müssen dann möglichst viele Aufgaben in einem bestimmten Zeitraum gelöst werden. Diese Zeitspanne ist ebenfalls einstellbar.
    Jede richtige Antwort erhöht die Punktzahl, jede Falsche erniedrigt diese.
    Nach Beenden der Stufe DE/ME gelangt man zum 'ENDE' Level.
  • In der Spielstufe 'ENDE' kann man die erreichte Punktzahl in einer Highscore-Liste zusammen mit einem Namen eintragen. Vorausgesetzt die erreichten Punkte sind hoch genug um einen Platz in dieser Bestenliste belegen zu können. Die Listen werden getrennt für die Stufen Mathe und Deutsch geführt.
    Durch Anwahl der Zielflagge (rechte Obere Ecke in der Menuleiste) kann man jederzeit aus einem Spiel-Level in dieses ENDE-Level gelangen.

Installation

 Das Spiel wurde mit der GTK-- Library erstellt. Deshalb sollte vor der Installation sichergestellt werden, ob diese vorhanden ist.
Das Installations-Skript sowie die Makefiles wurden mit den Tools autoconf und automake erzeugt.
Deshalb gliedert sich eine Installation in folgende Schritte:
  1. tar-File mit tar xvzf buzale.tgz auspacken
  2. cd buzale0.x
  3. ./configure
  4. make
  5. make install (als super-user)
Das Programm kann dann mit <Installationspfad>/buzale gestartet werden.
Vorher sollte man aber das Konfigurationsfile <Installationspfad>buzale_conf anpassen.
Dieses File wird, falls im HOME-Verzeichnis kein File mit dem Namen .buzale existiert, als Vorlage dorthin kopiert. Somit ist jede Spielkonfiguration User-Abhängig. Ist dies nicht gewünscht, kann man einfach einen Link auf ein zentrales File anlegen.
Zum Beispiel: ln -s <Installationspfad>buzale_conf ~/.buzale
Vor dem eigentlichen Spiel, gilt es jetzt aber kreativ zu sein und sich entsprechende Aufgaben in Form von Bilddateien (.xpm) und evtl. Sounddateien (.wav) zu erstellen.
Ein paar Beispiele liegen dem Paket schon bei. Die erstellten Sound.- und Bilddateien werden dann in das Konfigurationsfile eingetragen.

Konfiguration

 Das File ~/.buzale wird bei jedem Zugriff geöffnet und nach dem Lesen gleich wieder geschlossen. So ist es möglich in der Datei, während das Spiel läuft, Änderungen vornehmen zu können. (ES LEBE DER PLATTEN-CASH)
Ein Eintrag hat immer folgenden Syntax: Schlüsselwort = Parameter
Parameter, die aus mehreren Wörtern bestehen, können auch über mehrere Zeilen verteilt stehen. Das Ende der Zeile muß dann aber mit einem '\' abgeschlossen sein.
Jeder Eintrag liegt unter einer [Section].
[section1]
Schlüsselwort1 = Parameter
Schlüsselwort2 = Parameter
[section2]
Schlüsselwort1 = Parameter
Schlüsselwort2 = Parameter
Es sind folgende Sections vorgesehen:
[Allgemein]Allgemeine Einstellungen, die für das ganze Spiel gelten. Zum Beispiel: Farben, Schriftarten, Suchpfade
[START] Bilder und Sounddateien, die beim Spielstart (Intro) angezeigt werden sollen.
[DA]
..
[DE] 		Bilder und Sounddateien, die in den einzelnen Spielstufen im Zweig 'Deutsch' angezeigt werden sollen.
[MA]
..
[ME] 		Bilder und Sounddateien, für die Mathe-Aufgaben
[ENDE] Files in der der aktuelle Punktestand abgespeichert werden kann
(Highscore-Liste)
Wird unter einer Section ein erforderlicher Eintrag nicht gemacht,so sucht das Prgramm bis an das Dateiende nach einem passenden Eintrag. Damit ist es möglich Einträge die für mehrere Sections gleich sein sollen, nur in der letzten Section zu definieren. Zum Beispiel:
[DA]
[D1]
[DE]
name = Vogel Ofen
Die Bild.- und Sounddateien 'Vogel' und 'Ofen' sind für die Sections [DA], [D1], [DE] definiert.

z.B. zwei Bilder mit dem Namen 'Maus.xpm' -> Maus.1.xpm Maus.2.xpm
